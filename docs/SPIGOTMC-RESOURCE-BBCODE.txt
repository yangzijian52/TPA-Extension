[CENTER][SIZE=7][B]TPA Extension Documentation[/B][/SIZE][/CENTER]

[COLOR=#ff4d4d][B]Language notice:[/B][/COLOR] The SpigotMC resource page, documentation and support channel are English-only. Chinese-language support is not provided on SpigotMC.

[SIZE=5][B]Requirements[/B][/SIZE]
[LIST]
[*]Paper 26.2 only
[*]A Java runtime supported by Paper 26.2; the plugin itself is compiled to Java 21 bytecode
[*]EssentialsX is required and must be registered under the plugin name [ICODE]Essentials[/ICODE]
[*]Floodgate 2.x is optional and required only for native Bedrock forms
[*]Geyser is normally required for Bedrock clients but is not a direct plugin dependency
[*]Successfully tested with Java and Bedrock players on a live Paper 26.2 server
[/LIST]

[SIZE=5][B]Installation[/B][/SIZE]
[LIST=1]
[*]Stop the server completely.
[*]Install and configure EssentialsX.
[*]If Bedrock forms are required, install a compatible Geyser and Floodgate setup.
[*]Download [ICODE]tpa-extension-2.1.2.jar[/ICODE] from the GitHub Release or SpigotMC resource page.
[*]Place the jar in the server [ICODE]plugins[/ICODE] directory.
[*]Start the Paper 26.2 server.
[*]Confirm that TPAExtension and Essentials enable successfully in the console.
[*]Review [ICODE]plugins/TPAExtension/config.yml[/ICODE] and replace the default QuickMenu command if necessary.
[*]Restart the server after configuration changes.
[/LIST]

[SIZE=5][B]Player Commands[/B][/SIZE]
[LIST]
[*][ICODE]/btpa[/ICODE] - open the Java inventory GUI or Bedrock player-selection form
[*][ICODE]/btpa <name filter>[/ICODE] - open the selector with a case-insensitive partial player-name filter
[/LIST]

TPA Extension also observes the following canonical Essentials player commands to maintain its additional UI state:

[LIST]
[*][ICODE]/tpa <player>[/ICODE]
[*][ICODE]/tpahere <player>[/ICODE]
[*][ICODE]/tpaccept [player][/ICODE]
[*][ICODE]/tpdeny [player][/ICODE]
[*][ICODE]/tpacancel[/ICODE]
[/LIST]

[SIZE=5][B]Admin Commands[/B][/SIZE]
TPA Extension has no administrator command and no configuration reload command. Edit the configuration file and restart the server. Do not use Bukkit [ICODE]/reload[/ICODE]. Essentials administrator commands remain part of Essentials and are outside this plugin.

[SIZE=5][B]Aliases[/B][/SIZE]
[LIST]
[*][ICODE]/bedrocktp[/ICODE]
[*][ICODE]/btp[/ICODE]
[*][ICODE]/tpgui[/ICODE]
[*][ICODE]/tgui[/ICODE]
[/LIST]

Every alias opens the same selector as [ICODE]/btpa[/ICODE] and accepts the same optional name filter.

[SIZE=5][B]Permissions[/B][/SIZE]
[LIST]
[*][ICODE]tpaextension.btpa[/ICODE] - allows use of [ICODE]/btpa[/ICODE] and all aliases - default: all players
[*][ICODE]tpaextension.use[/ICODE] - declared for compatibility and future grouping - default: all players; the current build does not perform a separate runtime check for this node
[/LIST]

Players must also have the relevant Essentials permissions for [ICODE]/tpa[/ICODE], [ICODE]/tpahere[/ICODE], [ICODE]/tpaccept[/ICODE], [ICODE]/tpdeny[/ICODE] and [ICODE]/tpacancel[/ICODE]. TPA Extension does not bypass Essentials permission checks.

[SIZE=5][B]Main Request Flow[/B][/SIZE]
[LIST=1]
[*]A player opens [ICODE]/btpa[/ICODE] and selects another online player.
[*]The plugin dispatches the equivalent Essentials [ICODE]/tpa <player>[/ICODE] command as the requesting player.
[*]Essentials validates permissions and its own teleport-request rules.
[*]TPA Extension displays a cancel action to the requester and accept/deny actions to the target.
[*]Java players receive inventory or clickable chat controls; Floodgate players receive native Bedrock forms.
[*]Accept, deny and cancel actions are dispatched as the player through the matching Essentials command.
[*]The plugin removes its tracked UI state after response, cancellation, disconnect or expiry.
[/LIST]

[SIZE=5][B]Manual Command Flow[/B][/SIZE]
When players enter canonical Essentials [ICODE]/tpa[/ICODE] or [ICODE]/tpahere[/ICODE] commands in chat, TPA Extension observes the command and schedules the matching additional response UI. The plugin also observes canonical response and cancellation commands to close or remove tracked forms. Namespaced commands are supported after the namespace is removed; unrelated custom aliases are not observed.

[SIZE=5][B]Configuration File[/B][/SIZE]
The default file is generated at [ICODE]plugins/TPAExtension/config.yml[/ICODE]. It contains detailed Chinese comments and must remain UTF-8 encoded. YAML indentation must use spaces, not tabs. Existing configuration files are not overwritten automatically during upgrades.

[SIZE=5][B]Common Configuration Options[/B][/SIZE]
[CODE]gui:
  buttons:
    back:
      slot: 48
      material: ARROW
      name: "&aReturn to Main Menu"
      lore:
        - "&7Click to return to the server menu"
      commands:
        - "[command] quickmenu open"
    close:
      slot: 50
      material: BARRIER
      name: "&cClose Menu"
      lore: []
      commands:
        - "[close]"[/CODE]

[LIST]
[*][B]slot:[/B] valid Java GUI slot from 0 through 53. Slots 45 through 53 are recommended for footer controls.
[*][B]material:[/B] valid Paper 26.2 material name such as [ICODE]ARROW[/ICODE] or [ICODE]BARRIER[/ICODE].
[*][B]name and lore:[/B] support traditional ampersand color and formatting codes.
[*][B][command] <command>:[/B] executes the command as the player. A leading slash is optional.
[*][B][close]:[/B] closes the open Java inventory GUI.
[*][B]{player}:[/B] replaced with the clicking player's name in configured commands.
[*]Actions are executed from top to bottom and never bypass command permissions.
[/LIST]

[SIZE=5][B]Data Storage and Backup[/B][/SIZE]
TPA Extension does not create a database or persistent player-data file. Active UI request state is stored only in memory and contains requester UUID, target UUID, request type and creation time. It is discarded after response, cancellation, disconnect, expiry, plugin disable or server shutdown.

Back up [ICODE]plugins/TPAExtension/config.yml[/ICODE] before editing or upgrading. No player-data migration is required because the plugin does not own teleport or player-history data. Essentials data and configuration must be backed up according to the Essentials documentation.

[SIZE=5][B]Upgrading[/B][/SIZE]
[LIST=1]
[*]Stop the server.
[*]Back up [ICODE]plugins/TPAExtension/config.yml[/ICODE].
[*]Replace the previous TPA Extension jar with the new release jar. Do not keep multiple versions in the plugins directory.
[*]Compare the bundled [ICODE]config.yml[/ICODE] with the existing server file because Bukkit does not overwrite an existing configuration.
[*]Start the server and review the console for dependency or material warnings.
[*]Test one Java request and, if Floodgate is installed, one Bedrock request.
[/LIST]

[SIZE=5][B]Known Limits[/B][/SIZE]
[LIST]
[*]Paper 26.2 is the only tested server platform; Spigot compatibility is not claimed.
[*]The GUI selector sends [ICODE]/tpa[/ICODE] only. Use the manual [ICODE]/tpahere <player>[/ICODE] command for the reverse direction.
[*]The plugin adds UI and does not hide Essentials chat messages.
[*]Manual command enhancement recognizes canonical command names, not arbitrary third-party aliases.
[*]Only online players are listed and tracked.
[*]Tracked UI requests expire after approximately 30 seconds. Essentials controls its own request lifetime separately.
[*]Proactive Bedrock form closing depends on Floodgate exposing a compatible server-side close-form implementation. If unavailable, the plugin logs a warning and the teleport command can still be handled by Essentials.
[*]The default back button runs [ICODE]quickmenu open[/ICODE]. Change or remove this action if QuickMenu is not installed.
[*]There is no hot-reload command.
[/LIST]
